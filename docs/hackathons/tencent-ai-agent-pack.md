# 腾讯云 AI 智能体争霸赛提交包

## 项目名

AgentGuard CI：企业智能体上线前可靠性评测与治理平台

## 一句话介绍

在 AI 智能体真正操作企业系统之前，先自动测试它会不会越权、泄密、跳过审批、编造证据或造成不可逆操作。

## 中文参赛定位

腾讯云 AI 智能体争霸赛更适合把 AgentGuard CI 包装成中文企业级产品，而不是单纯的开发者工具。核心叙事应该是：

> 企业未来会部署大量客服、数据、流程、财务、人力、销售、安全与知识库智能体，但真正上线前需要一套可靠性评测和治理闸门。

AgentGuard CI 已经覆盖 13 类 Agent profile，其中 12 类是企业非代码智能体场景，包括浏览器/RPA、数据分析、客服、工作流、文档合规、邮箱日历、财务采购、HR 招聘、CRM 销售、安全 SOC、知识库检索和多智能体协作。

## 适配方向

- 中文 dashboard 和中文提交材料优先。
- 增加腾讯云部署/云函数/对象存储/日志服务的叙事。
- 把 Test Cloud evidence 改写成“企业智能体评测报告”和“上线审批证据包”。
- 增加中国企业场景：
  - 客服智能体误承诺大额退款。
  - 财务智能体越权审批采购单。
  - HR 智能体使用年龄/性别等敏感属性筛选候选人。
  - 知识库智能体被文档里的恶意提示词诱导。

## 推荐演示结构

1. 打开中文 dashboard，说明企业智能体的上线风险。
2. 展示 13 类 Agent 覆盖与 8 类失败模式雷达。
3. 运行 `npm run agentguard:agent-suite`。
4. 展示 12 个企业场景全部被路由到 review/block。
5. 展示一个财务或 HR 场景的报告。
6. 结尾强调：这不是普通测试平台，而是企业智能体的上线审批闸门。

## 为什么适合冲长期奖

- 用户在中国，参赛与沟通成本低。
- 截止时间更晚，能继续打磨真实功能。
- 企业智能体治理在中文市场也有强需求。
- 现有产品已经有中文 UI 和多场景基础。
